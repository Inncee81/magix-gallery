异常情况复现：
父子view分别存在校验逻辑
子view的以父view的id为前缀加下划线进行命名，如果 {{=viewId}}_inner 这种
子view双向绑定radio或者checkbox，并且挂载在data上数据为对象 info: {age: 1}
异常原因说明：
关于radio和checkbox的双向绑定，组件获取当前view中input[mxe="本viewid开头"]进行处理
复现情况中的子view id和magix本身处理mxe属性的规则一致，则父view处理时也获取到了子view的节点
由于子view的数据挂在info对象上，而父view上没有info对象
解决方案说明：
组件已兼容该情况
姓名
年龄
父view HTML Code
<form class="form form-4">
    <div class="form-item">
        <div class="form-label">姓名</div>
        <div class="form-content">
            <input class="input" placeholder="必填项" 
                value="{{:name{required:true}}}"/>
        </div>
    </div>
    <div class="form-item">
        <div class="form-label">年龄</div>
        <div class="form-content">
            <div mx-view="./error-inner" 
                id="{{=viewId}}_inner"
 ></div>
        </div>
    </div>
</form>
父view JS Code
let Magix = require('magix');
let $ = require('$');
let Form = require('@../mx-form/index');
let Validator = require('@../mx-form/validator');

module.exports = Magix.View.extend({
    tmpl: '@error.html',
    mixins: [Form, Validator],
    render() {
        let that = this;
        that.updater.digest({
            viewId: that.id,
            name: 'test'
        });
    }
});

子view JS Code
let Magix = require('magix');
let $ = require('$');
let Form = require('@../mx-form/index');
let Validator = require('@../mx-form/validator');

module.exports = Magix.View.extend({
    tmpl: '@error-inner.html',
    mixins: [Form, Validator],
    render() {
        let that = this;
        that.updater.digest({
            viewId: that.id,
            list: [{
                text: '10~20岁',
                value: 1
            }, {
                text: '20~30岁',
                value: 2
            }, {
                text: '30~40岁',
                value: 3
            }],
            info: {
                age: 2
            }
        });
    }
});
